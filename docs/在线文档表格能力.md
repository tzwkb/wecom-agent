# 在线文档与表格能力

更新于 2026-07-07。

## 实现策略

在线文档、在线表格、智能表格走 WecomTeam `wecom-cli`，本仓库只提供薄封装和 MCP 工具，不重写企微开放 API。

## 已实现

| 类型 | Python 封装 | MCP 工具 | 能力 |
|---|---|---|---|
| 普通文档 | `online.docs` | `wecom_doc_create`, `wecom_doc_write_markdown` | 创建文档、用 Markdown 覆写正文 |
| 本地文档 fallback | `online.local_docs` | `wecom_local_doc_read_path`, `wecom_local_doc_search` | 读取本机已缓存/已下载的文档文件 |
| 智能文档 | `online.docs` | `wecom_smartpage_create` | 创建智能文档/智能主页 |
| 在线表格 | `online.sheets` | `wecom_sheet_create`, `wecom_sheet_info`, `wecom_sheet_add_sub`, `wecom_sheet_delete_sub`, `wecom_sheet_append_row`, `wecom_sheet_update_range` | 创建表格、读取表结构、新增/删除子表、追加行、更新区域 |
| 智能表格 | `online.smartsheets` | `wecom_smartsheet_*` | 创建智能表格、读取子表/字段、管理子表/字段、增删改记录 |
| 自检 | `online.selfcheck` | CLI 命令 | 探测 `wecom-cli` 版本、授权状态、doc 子命令支持情况 |

## 当前不支持

| 能力 | 原因 |
|---|---|
| 读取线上普通文档最新版正文 | 当前 `wecom-cli 0.1.9` 未暴露 `get_doc_content`；本地 fallback 只能读已下载/已打开文件 |
| 读取在线表格单元格数据 | 当前 `wecom-cli 0.1.9` 未暴露表格数据读取命令；只有 `sheet_get_info` |
| 读取智能表格记录列表 | 当前 `wecom-cli 0.1.9` 未暴露 `smartsheet_get_records` |
| 按条件更新/删除智能表格记录 | 需要先读取记录拿 `record_id`，当前只能在已知 `record_id` 时更新/删除 |

## 写操作规则

所有线上写操作都需要显式 `confirmed=True`。确认前必须说明：

1. 目标文档或表格。
2. 写入、更新或删除的内容。
3. 是否可逆，以及是否需要用户人工清理。

## 本地缓存 fallback

当用户提供本地路径，或文档曾作为文件附件下载到本机时，可以用本地 fallback 解析：

```bash
PY=/opt/homebrew/bin/python3
$PY - <<'PY'
from online import local_docs
print(local_docs.search("报价"))
print(local_docs.read_path("/path/to/file.docx"))
PY
```

边界：

- 只读本地文件，不访问线上文档 API。
- 只能读取本机已下载或已打开过的文件。
- 结果可能落后于线上最新版。
- 在线表格/智能表格的结构化数据、字段类型、record_id 仍不能靠本地文件完整替代。

## 自检

```bash
PY=/opt/homebrew/bin/python3
$PY -m online.selfcheck
```

当前实测：

```text

```
