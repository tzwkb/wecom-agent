# 企业微信自建应用 API 联调 —— 需 IT 配合事项

> 用途：让运行在公司内部的 AI agent 能调用企业微信 API（发消息/查通讯录/读写在线文档）。
> 企业微信对国内主体**强制要求**「可信域名（须备案、主体=本公司）」+「企业可信IP」，故 A 线必须 IT 配合。
> **本地读取聊天记录（B 线）不需要本文任何东西**，已可独立使用。

## 背景信息

- 企业 CorpID：`ww...`（管理后台 → 我的企业 → 企业信息底部）
- 自建应用：WeCom-Agent　AgentId `1000000`
- 现象：调 API 报 `60020 not allow to access from your ip`；而配「企业可信IP」前，企微要求先验证一个**备案主体=琅科**的「可信域名」或「接收消息URL」。
- 已排除：cloudflare 等第三方临时域名 → 被企微「域名主体校验」拒绝（必须是琅科备案域名）。
- 可用域名：`langlobal-tech.com`（琅科备案）。现状：`www` 仅开了 http（nginx），**无 https**。

## 需要 IT 做的事

### 一、解锁「可信域名」（二选一，推荐方案 1）

**方案 1（推荐，最稳）——固定IP服务器 + 子域名 + HTTPS**
1. 提供/指定一台**有固定公网 IP** 的服务器（可复用现有 web 服务器），记其 IP 为 `SERVER_IP`。
2. 加 DNS A 记录：`wecom.langlobal-tech.com` → `SERVER_IP`。
3. 为 `wecom.langlobal-tech.com` 配 **HTTPS 证书**（Let's Encrypt 即可，企微校验要求 https）。
4. 该子域根目录可放静态文件（用于企微域名校验文件 `WW_verify_xxxxx.txt`，**文件名/内容由我方在企微后台生成后给你**）。

**方案 2（仅过门槛，较省事）——现有站点放校验文件**
- 在 `www.langlobal-tech.com` 根目录放一个 `WW_verify_xxxxx.txt`（内容我方提供），**且支持 https 访问**（现站无 https，需补证书）。

### 二、固定调用出口 IP（必须，否则发消息持续 60020）

- agent 调企微 API 的机器**出口公网 IP 必须稳定且在「企业可信IP」白名单内**。
- 实测本机（家宽/办公网 + 代理）出口 IP **会漂移**（同一机器先后呈现不同公网 IP），单 IP 白名单会失效。
- 解决：让 agent 的企微 API 调用**统一从一台固定公网 IP 的机器发出**（即方案 1 的 `SERVER_IP`，或一台固定 IP 的转发代理）。把该 `SERVER_IP` 给我方填入「企业可信IP」。

## 请 IT 回复

1. 可用的**固定公网 IP**（`SERVER_IP`）。
2. 能否加 `wecom.langlobal-tech.com` 解析并配 **https**。
3. 能否在该域名根目录放一个我方提供的校验文件。

> 拿到上述后，我方在企微后台生成校验文件 → 你放上去 → 我方点验证过「可信域名」→ 填 `SERVER_IP` 进「可信IP」→ A 线（发消息/文档/通讯录）即全部可用。
