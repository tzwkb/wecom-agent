# WeCom Agent Skill —— 开发计划与状态

> 目标：可分发 skill，让每位同事在**本地**用自己的 agent 操作/读取企业微信。
> 定位：**这是给 agent 用的 skill**——用户在本地直接指挥 agent，agent 通过本 skill 与企微交互。
> 更新于 2026-07-07。

## B 线：本地读取/解析 —— 当前开发重点

链路已通：`decrypt/macos/find_key_fast.py`(扫key) → `decrypt/macos/decrypt_wxwork.py`(解密) → `decrypt/export_wxwork.py`(导出)。当前重点改为稳定分发与边界补齐：

- [x] **B1. 发送者姓名**：`sender_id` → `Session.db` `USER`(RID→name) 解析为真名。
- [x] **B2. 消息类型分类**：文本、媒体、文件、卡片、系统、会议、文档链接已做基础渲染。
- [ ] **B3. 群成员名增强**：群聊里 `sender_id` 不在 USER 表时，继续解析 `CONV_MEMBER_DATA.member_bin` 或其他来源。
- [x] **B4. 增量监控 `monitor.py`**：用 key 定时解密 Info.db，按 `send_time` 取增量新消息。
- [x] **B5. 媒体导出**：图片/文件缓存导出已支持，同名文件会自动避让；语音可接转写。
- [x] **B6. 一键封装**：`find_key_fast → decrypt → export` 已串成 `read_wecom.py`，key 已存则跳过内存扫描。
- [x] **B7. Windows 端**：已有 `decrypt/windows/wecom_win.py` 与 PowerShell 一键入口。

风险：① key 随重新登录/换号可能变→失效重跑 find_key_fast；② 解密读取需对企微 adhoc 重签（破坏性，日常用需重装恢复，见 [[wechat_codesign_pitfall]]）。

## A 线：主动操作 —— 采用 WecomTeam CLI

不再继续推进自建应用 HTTP API 方案。主动操作直接复用 WecomTeam 官方能力，但可用范围取决于企业是否开放授权机器人对应权限。

- `wecom-cli todo`：已实测 `search_todo_userid` 与 `get_todo_list` 可用。
- `wecom-cli doc`：已新增 `online/` 封装和 MCP 工具，覆盖普通文档、在线表格、智能表格当前 CLI 支持的命令。
- `wecom-cli contact`：当前企业返回“不支持授权机器人「通讯录」使用权限”。
- `wecom-cli msg`：当前企业返回“不支持授权机器人「消息」使用权限”；不承担默认消息读取。
- `wecom-cli schedule`：当前企业返回“不支持授权机器人「日程」使用权限”。
- `wecom-cli meeting`：当前企业返回“不支持授权机器人「会议」使用权限”。

边界：消息读取全部默认走 B 线本地解密，包括历史、近期增量、离线搜索、统计、全量导出、已落地媒体定位。WecomTeam 的 `msg get_message` 不作为常规 fallback，仅在用户明确要求官方在线接口且本地读取不可用时应急使用。

## 里程碑
同事装上 skill → 本地对自己 agent 说自然语言 → agent 用 B 读/分析自己的企微历史聊天；主动操作由 WecomTeam `wecom-cli` 完成，不再依赖 IT 配自建应用域名/IP。
