# WeCom Agent Skill —— 开发计划与剩余步骤

> 目标：做成一个可分发 skill，让团队同事用各自 agent 操作企业微信（主动操作 + 读取聊天记录）。
> 本文记录距离"完整可分发"还差什么。更新于 2026-06-04。

| 能力 | 状态 | 说明 |
|---|---|---|

## A 线：主动操作（API）—— 剩余步骤

> 没有死结，确定能完成。卡在缺真实凭证。

- [ ] **A1. 真实凭证联调**：用户在企微管理后台建自建应用，拿 CorpID/Secret/AgentId
- [ ] **A2. 逐端点验证**：`gettoken` 已实测可达；contact/message/schedule/meeting/doc 各端点用真实凭证跑通，修正错误的端点/参数
- [ ] **A3. 补全封装**：按联调结果补 todo（待办）、smarttable（智能表格）等模块
- [ ] **A4. 权限文档**：明确每个功能需在后台开的权限 + IP 白名单要求（已在 README 起草，需联调确认）

---

## B 线：聊天记录读取（解密）—— 剩余步骤

> 核心已通（能 carve 出解密页、读到真实消息）。剩工程化与完整性。

- [ ] **B1. 每条消息精细结构化**：针对 `message_table` / `history_message_section_table` 写字段解析，做到每条「会话ID + 发送者 + 时间戳 + 消息类型 + 正文」逐条对应（当前是页级提取，字段未完全对齐）
- [ ] **B2. 全量历史捕获**：carve 依赖消息页当时在内存缓存。需脚本驱动「逐会话打开→触发加载→carve」循环，覆盖全部历史而非仅最近浏览
- [ ] **B3. 消息层 GCM 解密**：部分消息正文是 AES-256-GCM 二次加密（NID 901 key 已能捕获），需把 carve 出的 GCM 密文 blob + 捕获的 key 对应解开
- [ ] **B4. 一键自动化脚本**：把「重签(不用--deep) → 注入 → auto-login → carve → 解析」串成一条命令，用户跑一次即导出全部消息
- [ ] **B5. 媒体消息**：图片/文件/语音的解密与导出（语音可复用现有 transcribe）
- [ ] **B6. Windows 端**：当前只验证 macOS；Windows 企微解密路径需另做

## 优先级建议

1. **先做 B1+B4**（读取线工程化）—— 这是已攻克、最有价值、用户最想要的，做成一键导出
2. **再做 A1-A2**（主动操作联调）—— 需用户提供凭证，确定能成
3. **最后 C**（打包分发）—— 两能力稳定后整合

## 关键风险/依赖
- B 线依赖 macOS 重签名（破坏性，需重装恢复日常使用）—— 分发时要让同事知情
- A 线依赖用户是企微管理员（建自建应用）
- 重签**必须不用 --deep**（否则登录崩，已踩坑确认）
- Windows 端是未知区（B6）

## 里程碑定义（何为"完整"）
- 同事装上 skill，配好凭证 → 对自己 agent 说自然语言 → 既能在企微发消息/查通讯录/建日程（A），也能一键导出+检索自己的聊天记录（B），全程不需懂技术细节。
