# 企业微信自建应用配置教程（wecom-agent 用）

> 目标：拿到 skill 所需的**全部凭证**并配好**实时接收回调**，解锁「发消息 / 读写文档 / 实时接收+自主处理」。
> 本地「读取聊天记录」走解密线，**不需要**本教程的任何东西。
> 前提：你是本企业的**企业微信管理员**。官方文档：[开发者中心](https://developer.work.weixin.qq.com/document/path/90664)。

---

## 一图看清需要哪些凭证

| 能力 | 需要的凭证/配置 |
|---|---|
| 发消息、查通讯录、读写文档（主动调 API） | **CorpID** + 应用 **Secret** + **AgentId** + **企业可信IP** |
| 实时接收消息 + 自主处理 | 上面全部 + 接收消息的 **Token** + **EncodingAESKey** + **公网回调URL** |

---

## 步骤 1：创建自建应用 → 拿 AgentId / Secret

1. 浏览器登录管理后台 <https://work.weixin.qq.com/wework_admin>（管理员扫码）。
2. 顶部 **应用管理** → 下方 **自建** → **创建应用**。
3. 填表单（英文界面对照）后点 **Create / 创建应用**：
   - **App Name**（应用名称）：随便起，如 `AI Agent`。
   - **App Logo / About this App**：可留空（可选）。
   - **Allowed users**（可见范围）：选**你自己**（测试期必须含自己——决定 app 能给谁发消息、能在谁的会话里操作，之后可改）。
   - **Create a Mini Program/app quickly based on an existing Mini Program**：**忽略/不要选**。那是"基于已有小程序快速创建"的入口；我们要的是用**服务端 API** 发消息/文档的普通自建应用，不是小程序。
4. 进入该应用详情页，记下：
   - **AgentId**（应用 ID，纯数字）。
   - **Secret**：点「查看」→ 会推送到你企业微信里的「企业微信团队」，复制那串。

## 步骤 2：拿企业 ID（CorpID）

**我的企业** → **企业信息** → 拉到页面最底部 → **企业ID**，复制。

## 步骤 3：开权限（按需）

- **读通讯录**（可选，找人/查部门用）：自建应用读通讯录受隐私限制。两条路：①应用「可见范围」内成员可被读到基本信息；②需要全量时走 **管理工具 → 通讯录同步**，开启后另给一个「通讯录 Secret」。注意 2022-08 起 `user/list`(部门成员详情) 对通讯录同步新IP下线，改用 `user/list_id`（wecom.py 已适配）。
- **在线文档（wedoc）**：确保企业已开通「文档/微盘」。文档接口用应用 access_token 调用；新建/读取的文档归该应用。无需额外开关，但企业须未禁用文档。

## 步骤 4：配「可信域名 / 接收URL」+ 企业可信IP（否则发消息报 60020）

> 规则：**配企业可信IP 前，必须先设置「可信域名」或「接收消息服务器URL」之一**（[来源](https://blog.csdn.net/weixin_45385457/article/details/132278442)）。最省事是直接做步骤 5 的接收URL，就同时满足了前置条件。

- 应用详情页拉到底 → **企业可信IP** → 添加**你运行 agent 的机器的出口公网IP**。
  - agent 跑在本 Mac（家/公司网）→ 填你的宽带出口公网IP（浏览器搜「我的IP」）。IP 会变则需用固定出口（VPS/专线）。
  - agent 跑在 VPS → 填 VPS 公网IP。
- 不配或配错 → 调 API 返回 `errcode 60020 not allow to access from your ip`。

## 步骤 5：配「接收消息」回调（实时接收 + 自主处理必需）

> 这是「实时收到消息→自动分析→自动回复/写文档」的入口。([官方:接收消息](https://developer.work.weixin.qq.com/document/path/90238))

1. 先准备一个**公网可达的 HTTPS 地址**指向本机服务（三选一）：
   - **cloudflared**（推荐，免费免备案）：`cloudflared tunnel --url http://localhost:8000` → 得到一个 `https://xxx.trycloudflare.com` 临时域名。
   - **ngrok**：`ngrok http 8000` → `https://xxx.ngrok-free.app`。
   - **VPS + 域名**：把回调反代到本机（或直接在 VPS 跑 agent）。
2. **先启动接收服务**（我会给你 `recv_server.py`，它实现 URL 验证），再回后台填写。
3. 应用详情页 → **接收消息** → **设置API接收** → 填三项：
   - **URL**：上面的公网地址（如 `https://xxx.trycloudflare.com/wecom`）。
   - **Token**：点「随机获取」或自填一串。
   - **EncodingAESKey**：点「随机获取」（43 位）。
4. 点**保存**：企业微信会向该 URL 发一个 GET（带 `msg_signature/timestamp/nonce/echostr`）做验证；服务端须用 Token+EncodingAESKey 校验签名、解密 `echostr` 并原样返回明文。**服务没在跑/验证不过 → 保存失败**。`recv_server.py` 已实现这一步。
5. 保存成功后：成员在应用会话里发的消息会 POST 推到该 URL（加密 XML）；服务端解密拿到 `发送者/文本/会话`，**5 秒内**要么直接在响应里回包、要么走主动 API 回复（超时不回，企微重试 3 次）。

## 步骤 6：把凭证给我（或填 config.json）

`wecom-agent/config.json`（已在 .gitignore，不进 git）：

```json
{
  "corpid":  "ww................",
  "secret":  "应用Secret",
  "agentid": "1000002",
  "recv_token": "步骤5的Token",
  "recv_aeskey": "步骤5的EncodingAESKey(43位)",
  "public_url": "https://xxx.trycloudflare.com/wecom"
}
```

只做发消息/文档：前 3 项即可。要实时接收+自主处理：全填。

## 注意事项

- **5 秒响应**：被动回包要快；复杂处理（调大模型、写文档）应先 ACK 再异步用主动 API 回复，避免超时重试。
- **加解密**：回调用 AES-256-CBC + 企业 `corpid` 校验，签名用 `sha1(token,timestamp,nonce,encrypt)`，`recv_server.py` 内置（移植官方 WXBizMsgCrypt）。
- **安全**：Secret/Token/AESKey 等同密码；只填 config.json（gitignore）；可信IP 收紧到必要范围；agent 自主回复/写文档前建议留人工确认开关。
- **可见范围**：应用只能操作可见范围内的成员/会话；测试期把自己加进去。
